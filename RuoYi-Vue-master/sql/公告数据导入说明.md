## 数据字段说明

每条公告包含以下字段：
- `notice_title`: 公告标题
- `notice_type`: 公告类型（1=通知，2=公告）
- `notice_content`: 公告内容（HTML格式）
- `status`: 状态（0=正常，1=关闭）
- `create_by`: 创建人（admin）
- `create_time`: 创建时间（当前时间）
- `update_by`: 更新人（admin）
- `update_time`: 更新时间（当前时间）
- `remark`: 备注
- `community_id`: 小区ID（NULL，可根据实际情况修改）

## 导入步骤

### 方法一：使用数据库管理工具
1. 打开 Navicat、MySQL Workbench 或其他数据库管理工具
2. 连接到你的数据库
3. 选择 RuoYi 数据库
4. 打开 `sql/insert_notice_data.sql` 文件
5. 执行SQL语句

### 方法二：使用MySQL命令行
```bash
mysql -u用户名 -p密码 数据库名 < sql/insert_notice_data.sql
```

### 方法三：使用MySQL客户端
```sql
source /path/to/sql/insert_notice_data.sql;
```

## 注意事项

1. **小区ID字段**：当前 `community_id` 设置为 NULL，如果需要关联特定小区，请在导入前修改SQL文件中的 `community_id` 值
2. **创建时间**：所有公告的创建时间使用 `NOW()`，导入后会自动设置为当前时间
3. **状态字段**：所有公告状态默认为 0（正常），表示公开发布
4. **内容格式**：公告内容使用HTML格式，支持富文本显示

## 验证导入

导入后，可以执行以下SQL验证：
```sql
SELECT notice_id, notice_title, notice_type, create_time
FROM sys_notice
ORDER BY create_time DESC
LIMIT 15;
```

## 小程序显示效果

导入后，小程序首页将：
1. **顶部轮播图区域**：显示最新5条公告（可点击查看详情）
2. **底部小区公告模块**：显示最新10条公告（带标题、日期、缩略图占位符）
3. **点击"查看全部"**：跳转到完整的公告列表页

## 自定义修改

如需修改公告内容，可以：
1. 直接编辑 SQL 文件中的公告标题和内容
2. 修改 `notice_type` 字段（1=通知，2=公告）
3. 添加 `community_id` 关联特定小区
4. 调整 `create_time` 为特定日期

## 后续维护

- 新增公告：通过物业管理后台添加
- 修改公告：在后台编辑现有公告
- 删除公告：将 `status` 改为 1（关闭），或直接删除记录
